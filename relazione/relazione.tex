\centerline{\bfseries{Sommario}}
Il progetto consiste nella realizzazione di due software che implementano la compressione delle immagini basata su DCT.

Nella prima parte viene implementato l'algoritmo DCT2, confrontando successsivamente i tempi di esecuzione con quello implementato dalla libreria numpy.

Per la seconda parte il programma permette all'utente di scegliere da filesystem un'immagine bitmap in toni di grigio, applicare la compressione secondo i parametri specificati e visualizzare affiancate l'immagine originale e compressa.

\section{Introduzione}
La funzione dct2 calcola la trasformata discreta del coseno 2D di una matrice 2D, tipicamente un'immagine. Questa trasformata converte i dati spaziali in componenti di frequenza. \`E una tecnica fondamentale per la compressione di immagini, alla base dello standard JPEG. 

L'operazione \`e separabile: equivale a applicare una DCT 1D prima sulle righe e poi sulle colonne (o viceversa). I coefficienti ottenuti permettono di ricostruire l'immagine originale tramite la trasformata inversa (idct2). Durante la compressione, i coefficienti ad alta frequenza, che risultano essere meno percepibili all'occhio umano, possono essere eliminati per ridurre la dimensione del file con una perdita di qualità limitata.

\section{Implementazione DCT / DCTN}
L'idea che sta alla base del calcolo della \texttt{DCT} \`e la trasformazione da base canonica in uno spazio vettoriale a quella dei coseni.\\

\subsection{Implementazione DCT1}
Come fattore di normalizzazione viene usato \texttt{1/sqrt(N)} per il primo coefficente, per i restanti invece \texttt{sqrt(2/N)}.\\
In fase di inizializzazione viene creato un vettore di output \texttt{y}. di lunghezza pari alla dimensione del vettore, pieno di zeri.

\begin{lstlisting}[language=Python]
N = len(v)
y = np.zeros(N, dtype=float)
\end{lstlisting}

Successivamente viene calcolato il coefficente \texttt{k} per ogni posizione nel seguente modo:


\begin{lstlisting}[language=Python]
if k == 0:
	Ck = 1.0 / math.sqrt(N)
else:
	Ck = math.sqrt(2.0 / N)
	
for i in range(N):
	ak = math.cos((math.pi * k * (2*i + 1)) / (2 * N)) * v[i]
	sum_cos += ak
	y[k] = Ck * sum_cos
\end{lstlisting}

Dove \texttt{Ck} \`e il fattore di normalizzazione, \texttt{ak} \`e la base della DCT, che rappresenta le diverse frequenze, la quale viene aggiunta alla somma dei coseni.\\
In questo modo non si ha pi\`u informazione locale come nella base canonica ma si ha informazione riguardante tutto il vettore.\\
Il risultato \`e quindi il vettore \texttt{y} di coefficienti che rappresenta l'input in termini di componenti di frequenza del coseno.

\subsection{Implementazione DCT2}
Per calcolare la funzione \texttt{DCT2} \'e possibile iterare su righe e colonne la \texttt{DCT1}.\\

\begin{lstlisting}[language=Python]
# DCT1 for columns
for j in range(N):
	y_m[:,j] = dct(m_in[:,j])

# DCT1 for rows
for j in range(N):
	y_m[j,:] = np.transpose(dct(np.transpose(m_in[j,:])))
\end{lstlisting}

Il primo passaggio \`e scorrere ogni colonna della matrice originale applicando la funzione dct1, salvando il risultato nella corrispondente colonna.\\
Successivamente si applica lo stesso passaggio, alla matrice ottenuta precedentemente, ma sulle righe.

\subsection{Implementazione}
Il file \texttt{benchmark.py} implementa un semplice benchmark sulla funzione \texttt{dct2} implementata, su matrici di dimensione NxN. con N che ha range da 10 a 200 in step da 10. Confrontandola con quella della libreria Python \texttt{numpy}

\lstinline|n_values = np.arange(10, 201, 10)|
\\
\noindent
Per poi mostrare i risultati a schermo tramite la libreria \texttt{matplotlib}.


\begin{lstlisting}[language=Python]
for n in n_values:
	x = np.random.randint(0, 255, (n, n)).astype(float) - 128

	start_t = time.perf_counter()
	_ = dctn(x, type=2, norm='ortho')
	end_t = time.perf_counter()
	scipy_times.append((end_t - start_t) * 1000) # Convertiamo in ms

	start_t = time.perf_counter()
	_ = utils.dct2(x)
	end_t = time.perf_counter()
	custm_times.append((end_t - start_t) * 1000) # Convertiamo in ms

	print(f"Test completato per N={n}")

return np.array(scipy_times), np.array(custm_times)
\end{lstlisting}

La funzione esegue quindi i seguenti step per ogni valore nel range definito:

\begin{enumerate}
	\item Genera una matrice quadrata $n \times n$ di valori interi casuali tra 0 e 255 (simulando pixel di un'immagine)
	
	\item Viene applicata la funzione \texttt{dctn} della libreria, tenendo tracia del temp di esecuzione, usando i parametri \texttt{norm='ortho'} che applica una normalizzazione ortonormale. e \texttt{type=2} che utilizza lo standard per la compressione.
	
	\item Viene misurato il tempo della funzione \texttt{dct2} implementata non dalla libreria sulla stessa matrice
	
	\item Infine vengono salvati i tempi di esecuzione per essere successivamente mostrati
\end{enumerate}

\subsection{Implementazione}

Una volta importata l'immagine Bitmap e convertita in toni di grigio, viene compressa e salvata, rappresentata in bytes, in un buffer, quindi mostrata a fianco dell'originale.

\begin{lstlisting}[language=Python]
compressed = jpeg.compress_image(img, F, d)
buf = BytesIO()
compressed.save(buf, format="BMP")
return buf.getvalue()
\end{lstlisting}

La logica di compressione \`e implementata nel file \texttt{jpeg.py}. In particolare la funzione \lstinline|compress_image| \ref{lst:compress-image} \`e quella che implementa la compressione dato un oggetto \texttt{Image.Image}.

\begin{lstlisting}[caption={compress\_image}, escapeinside={(*}{*)}, language=Python, label={lst:compress-image}]
def compress_image(img: Image.Image, F: int, d: int) -> Image.Image:
	"""Compress a grayscale PIL image with DCT2 block compression using multiple CPU cores."""
	A = np.array(img.convert("L")) # convert to grayscale (*\label{line:save_a}*)
	n_y, n_x, blocks = block_splitter(A, F) (*\label{line:split}*)
	core = os.cpu_count() or 1
	max_cores = max(1, core - 2) #lasciamo 2 core liberi per il sistema e altre operazioni( se meno di cpu.count() < 4 core allora ne usa 1)
	with ProcessPoolExecutor(max_workers=max_cores) as executor:
		compressed_iter = executor.map(
			compress_block, 
			blocks, 
			[d] * len(blocks)
		)
		compressed = list(compressed_iter)
	rows = [np.hstack(compressed[by * n_x : (by + 1) * n_x]) for by in range(n_y)] (*\label{line:hstack}*)
	return Image.fromarray(np.vstack(rows) (*\label{line:vstack}*)
\end{lstlisting}

La funzione inizializza \texttt{A} come array numpy dell'immagine passata (\ref{line:save_a}), divide quindi la matrice (array \texttt{A}) in blocchi da $F \times F$ (\ref{line:split}) ed esegue la compressione su ognuno di essi, parallelizzando l'operazione.\\
Infine ricostruisce l'immagine dai blocchi compressi, prima chiamando la funzione numpy \texttt{hstack} (\ref{line:hstack}), che concatena gli array orizzontalmente, e successivamente \texttt{hstack} (\ref{line:vstack}), che invece concatena gli array in verticale, ottenendo cosi l'immagine completa.

La funzione \texttt{block\_splitter} \ref{lst:block-splitter} implementa la logica di suddivisione della matrice rappresentante l'immagine in blocchetti, allineati cominciando dal'alto a sinistra.
\begin{lstlisting}[caption={block\_splitter}, escapeinside={(*}{*)}, language=Python, label={lst:block-splitter}]
def block_splitter(A: np.ndarray, F: int) -> tuple[int, int, list[np.ndarray]]:
	"""Split a 2-D array into FxF blocks (top-left aligned, remainder discarded).
	Returns (n_blocks_y, n_blocks_x, list_of_blocks).
	"""
	h, w = A.shape
	n_y, n_x = h // F, w // F

	#crop the input array A to ensure its dimensions are multiples of F.
	A = A[: n_y * F, : n_x * F].astype(float)

	#extract the FxF block from the cropped array A for each block position (by, bx).
	return n_y, n_x, [
		A[by * F : (by + 1) * F, bx * F : (bx + 1) * F]
		for by in range(n_y)
		for bx in range(n_x)
	]
\end{lstlisting}
